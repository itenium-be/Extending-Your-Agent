## If you only do three things

1. **Connect your tracker.** You're on the Microsoft stack, so wire up the **Azure DevOps MCP** (#2). If your shop is Atlassian, use **Atlassian Rovo MCP** (#1) instead — it's remote OAuth, zero install. Either one lets Claude read/write work items and wiki/Confluence pages directly, so "draft acceptance criteria on PROJ-123" or "create a wiki page from this spec" become one-liners.
2. **Add one skill pack.** Install **phuryn/pm-skills** (#4) — 68+ skills (PRDs, user stories, OKRs, opportunity-solution trees) and the most popular of the bunch (21k★). It encodes real frameworks (Torres, Cagan) instead of generic prose.
3. **Add a diagram skill.** **drawio-skill** (#6) for BPMN/process flows you can hand to stakeholders, or **claude-mermaid** (#7) if you live in markdown/wiki and want diagrams-as-code with live preview.

## Notes

- **Skill vs plugin vs MCP server vs subagent:** *MCP servers* connect Claude to external systems (Jira, ADO, GitHub). *Skills* are markdown know-how packs that teach Claude a method (write a PRD the right way). *Subagents* are role-scoped personas Claude delegates to. *Plugins* bundle any of the above for one-command install.
- On-prem Jira/Confluence (Server/Data Center)? Rovo is Cloud-only — use the community **[sooperset/mcp-atlassian](https://github.com/sooperset/mcp-atlassian)** instead (supports DC + Cloud). See FULL_REPORT.md.
