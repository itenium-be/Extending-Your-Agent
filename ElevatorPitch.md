# Extending-Your-Agent

## Abstract

**Configure → Extend → Compose → Ship.** You've met skills, agents, and hooks — now make Claude Code your own. Turn the knobs (settings, context engineering, models, permissions), author each primitive around one principle — the description *is* the interface — then compose them into installable plugins you pull from a marketplace and ship to your team. MCP gets its own session later.

## Target Audience

Developers who want to go past the defaults: turn the knobs in `settings.json`, write their own skills, agents, and hooks, and package a team playbook as a plugin. Fourth in the itenium AI deep-dive series — you know the pieces exist; here's how to build with them.

## Key Takeaways

- Configure the harness to fit you: context engineering, models & effort, permissions & sandbox — via `/update-config`.
- The description is the interface — the trigger that activates a skill and the dispatch decision for a subagent.
- Pick the right primitive: repeatable workflow → skill, isolated context → agent, determinism → hook.
- Compose skills, agents, and hooks into one installable plugin, and pull others from the marketplaces.

## Session Format

90 minutes — virtual, deep-dive.
